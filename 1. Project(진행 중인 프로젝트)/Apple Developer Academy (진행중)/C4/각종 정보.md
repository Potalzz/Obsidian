
## Table Tabular Regressor
### Tabular Regressor는 무엇인가 ?
**Tabular = 테이블**

**Regressor = 숫자를 예측하는 모델**


### 언제 사용할 수 있는가 ?
**예시**
- 주택 가격 예측 (위치, 집 컨디션 등등의 조건을 넣어서)
- 자동차 보험료 예측
- 배달 시간 예측


### 어떻게 사용하나 ?
**데이터 전처리 과정이 필요하다**

로우데이터를 가공해야 한다.

#### 할 수 있는 것
**수치형 연속 값 예측**
- 주택 가격, 매출 예측, 온도 예측 등

**회귀에 적합한 알고리즘 제공**
- Boosted Tred

**클래스가 아닌 수치값 타겟 지정 가능**
- 분류와는 달리 label은 연속적인 수치 값

#### 할 수 없는 것
**복잡한 사용자 정의 전처리 불가**
- 수식, 조건 처리, 데이터 변환 로직 등은 사전 ㄴ수동 처리 필요

**하이퍼파라미터(바꿀 수 있는 파라미터) 수동 튜닝 불가**
- 알고리즘 내부 옵션 조정 거의 불가

**모델 내부 구조 시각화 불가**
- 트리 구조, 가중치 계수 직접 확인 어려움

**외부 평가 함수 적용 불가**(모델이 만들어지고 유용한지 평가)
- 기본 지표 외 사용자 정의 평가 함수는 지원 안됨


#### Target & Features 선택 기준
**목표(Target)와 직접적인 관련성**
- 타겟값과 상관관계각 높은 변수 선택

**독립성**
- 서로 중복되지 않고 의미가 다른 변수 선택

**데이터 타입과 수치적 의미**
- 범주형 변수는 인코딩 처리되어야 하고, 수치형 변수는 정규화 필요할 수 있음

**피처 개수 균형**
- 너무 많은 변수는 모델 복잡도 증가


#### 알고리즘
**Random - Forest - 숲 속의 다수결**
- 여러 개의 결정 트리를 앙상블하여 예측 정확도와 일반화 성능을 높임
- 과적함에 강함: 여러 트리의 평균을 사용하므로, 단일 Decision Tree보다 일반화 성능이 우수

**Boosted Tree - 계속 배우는 나무**
- 이전 모델의 오차를 보완해가며 학습하므로 복잡한 문제에서도 높은 정확도
- 불균형 데이터에 강함

**Decision Tree - 질문으로 내려가는 나무**
- 모델 해석력 탁월: 각 분기 조건이 명확하여 "왜 이런 예측이 나왔는가?"를 설명하기 유용
- 학습 속도가 빠름: 간단한 구조 덕분에 소규모 데이터나 빠른 실험에 적합

**Linear Regression - 직선 하나로 예측**
- 선형 문제에 적합: 타겟과 피처 간 관계가 선형일 경우 매우 효과적
- 매우 빠른 학습 및 예측이 가능하여 큰 데이터셋에도 부담 없이 사용
