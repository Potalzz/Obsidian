**npm install 에러**
- multer-storage-cloudinary 패키지를 설치하려는데, 관련 패키지인cloudinary는 업데이트가 최신 버전인데에 반해, 옛날 버전이라서 버전 충돌이 일어났다. cloudinary를 다운그레이드하고 설치하니 문제 해결. 

**git add 에러**
- auto crlf해도 계속 에러가 발생해서 그냥 **git config —global core.safecrlf false**로 경고 꺼줌. 
- 계속 에러가 뜨는 이유 : https://velog.io/@dailylifecoding/Git-Bash-Window-git-configure-global-core.autocrlf-true-but-still-warning

**Yelp Camp 배포 중 발생한 에러(mongoDB connection Error)**
- mongo connectinon 타깃 주소를 mongo url로 바꿔야 하는데, 로컬 url로 해놓아서 생긴 문제였음.

**Yelp Camp 배포 중 발생한 에러 (Render No such file directory)**
- boilerplate를 불러오지 못한다면서 위 내용의 에러가 떴다. 스펠링 자체에 관한 에러메시지이지만, 파일 네임을 건드린 적이 없다고  생각하고 계속해서 헛다리를 짚었다. 알고보니 boilerPlate의 P를 index파일에서 소문자 p로 불러들이고 있어서 생긴 에러였다.
- 무작정 비슷한 사례를 검색해서 해결하려 하지 말고 **에러 메시지의 의미**와 **내 코드**에 집중하자. 에러가 발생한 로직을 파악하고 이해하면 쉽게 해결할 수 있다. 해결하기 어려울 거라는 생각을 버리자.
