### 컴퓨팅 사고

**컴퓨팅 사고의 정의**
컴퓨터가 효과적으로 수행할 수 있도록,
문제를 정의하고 그에 대한 답을 기술하는 것을 포함한 사고과정.

**컴퓨팅 사고 기법**
분해 : 복잡한 문제를 좀 더 쉽게 해결 가능하도록 세분화 시키는 것
알고리즘 : 어떠한 문제를 해결하기 위해 정해진 일련의 절차나 방법을 공식화한 형태로 표현하는 것.
추상화 : 문제를 간략화 시키는 것. 데이터 수집 -> 가시화 -> 간략화 -> 변수
패턴 인식 : 데이터 안에서 특정 패턴을 찾아내는 것

**컴퓨터 사고 기법 4가지의 과정**
문제를 **분해** **=>** **추상화**를 통해 변수를 만들어내고 **=>** 변수로부터 **패턴**을 찾아내고 **=>** 찾아낸 패턴으로부터 **알고리즘**을 구성.
**분해 => 추상화 => 패턴인식 => 알고리즘**을 거치면 최종적으로
**순서도**나 **상태 전이도**가 나오게 된다.

**순서도** : 다이어그램의 종류 중 하나로 여러 종류의 상자와 이를 이어주는 화살표로 명령의 순서를 보여
**상태 전이도** : 다이어그램의 종류 중 하나로 여러 종류의 **상태**와 이를 이어주는 화살표를 이용해 **상태의 변화과정**을 보여주는 프로세스

결과적으로 **컴퓨팅 사고**란 ?
- 어떤 문제를 해결하기 위해 **분해, 추상화, 알고리즘, 패턴인식**의 컨셉을 적용하여 **순서도** 혹은 **상태전이도** 형태의 결과물을 도출하는 것.

### 일반적인 사고와 컴퓨팅 사고

**일반적인 사고**
- 동시 다발적 생각
- 빠른 의사 결정
- 블랙박스 의사 결정 과정

**컴퓨팅 사고**
- 순차적 생각
- 느린 의사 결정
- 의사 결정 과정이 명확함

**컴퓨팅 사고 기법 핵심**
어떤 패턴이 있다면 그래프로 만들 수 있고, 그래프로 변환시킬 수 있다면 프로그래밍 할 수 있다.
